# Agent 技能集：项目自动化文档生成 Skills

本文为 **howie\-daily\-helpers** Monorepo 项目专属 **AI Agent Skills 能力定义文件**。用于告知 AI Agent 拥有「源码扫描、自动生成 VitePress 文档、自动更新站点配置」全套自动化能力，严格按照本技能规则执行，支持增量更新、自动适配新增文件/新增子包，全程无人值守。

## 一、技能归属与项目架构

### 技能适用项目

howie\-daily\-helpers 基于 **monorepo \+ pnpm** 架构，区分源码子包与 VitePress 文档应用，目录结构固定，AI Agent 需基于该结构执行文档生成技能。

以下为项目标准参考目录结构，仅约定顶层目录 packages/、apps/ 职能与文件映射逻辑，子包数量、模块文件不固定，支持自由扩展新增。

```text
howie-daily-helpers/
├── packages/                      # 源码子包目录（技能扫描根目录、NPM发包开发目录）
│   ├── core/                      # 纯TS核心工具包（无框架依赖）
│   │   ├── src/                   # 工具模块源码
│   │   │   ├── index.ts
│   │   │   ├── logger.ts
│   │   │   ├── is.ts
│   │   │   └── string.ts
│   │   ├── test/
│   │   ├── package.json
│   │   └── tsconfig.json
│   ├── dom/                       # DOM 工具子包
│   └── shared/                    # 内部共享私有子包
├── apps/
│   └── docs/                      # VitePress 文档站点（技能输出目录）
│       ├── .vitepress/
│       │   └── config.mts         # 站点配置自动更新目标文件
│       └── markdown/              # AI 自动生成文档输出根目录
│           └── [子包名]/          # 自动对齐 packages 子包结构
├── package.json
├── pnpm-workspace.yaml
├── tsconfig.base.json
└── vitest.config.ts
```

## 二、强制工程开发规范（开发者约束）

为统一项目发包规范、保证 monorepo 工程整洁，强制执行以下开发规则，同时作为 AI Agent 识别合法源码包的判定依据。

- **NPM 发包唯一目录约束**：所有需要打包、发布到 NPM 的公开工具包、可复用业务工具包，**`packages/`****必须且只能在  目录下进行开发**。

- **目录职能隔离规则**：
        

    - `packages/`：存放可发包、可导出、可被外部引用的通用 TS 工具包，为唯一合法发包源码目录。

    - `apps/`：仅存放可独立运行的应用项目（文档站点、业务应用等），**禁止在 apps 目录开发任何 NPM 发包源码**。

- **Agent 识别规则**：仅对 `packages/` 目录下的子包执行文档生成、配置同步能力，`apps/` 目录文件不参与工具文档自动化生成逻辑。

## 三、Agent 核心技能定义（必须严格执行）

本技能集包含四大核心能力：**源码映射生成文档**、**标准化 MD 文档生成**、**子包汇总首页生成**、**VitePress 配置自动同步**。

### Skill 1：源码与文档目录自动映射能力

AI Agent 自动扫描 `packages/` 目录下所有子包、所有 `*.ts` 工具模块，实现结构一对一映射生成文档。

- **源码输入路径**：`packages/[子包名]/src/[模块名].ts`

- **文档输出路径**：`apps/docs/markdown/[子包名]/[模块名].md`

- **技能规则**：完全对齐目录结构，新增子包、新增 ts 文件自动对应生成 md 文件，删除源码自动清理对应文档

- **示例**：`packages/core/src/logger.ts` ➜ `apps/docs/markdown/core/logger.md`

### Skill 2：标准化 MD 文档生成能力

所有自动生成的 Markdown 文件严格适配 VitePress 站点，根据 TS 源码语义自动解析生成完整文档。

- 自动识别模块功能、工具函数、入参、出参、注释说明

- 自动生成：模块简介、函数说明、参数表格、返回值说明、代码示例、使用场景

- 遵循 VitePress  markdown 语法规范，支持标题层级、代码高亮、链接跳转

- 支持增量更新：源码变更后自动覆盖更新对应文档内容

### Skill 3：子包汇总首页 index\.md 生成能力

针对每一个 `packages` 子包，自动生成分组总览首页，作为文档分组入口。

- 自动生成文件：`apps/docs/markdown/[子包名]/index.md`

- 页面作用：展示当前子包所有工具模块、所有函数清单

- 自动生成内部跳转链接，点击可跳转对应模块详情文档

- 子包文件变更时，自动刷新 index\.md 目录与链接

### Skill 4：VitePress 配置自动同步能力

文档生成/更新/删除后，自动同步修改 `apps/docs/.vitepress/config.mts`，无需人工改配置。

- 自动更新 **nav 顶部导航**：新增子包自动追加导航入口

- 自动更新 **sidebar 侧边栏**：严格根据 markdown 目录生成层级菜单

- 层级规则：子包 index\.md 为一级菜单，各模块 md 文件为二级子菜单

- 保留原有自定义配置，仅自动维护文档路由与菜单栏，不破坏原有配置

- 自动清理失效菜单：删除文档后自动移除无效配置项

### Skill 5：自动生成对应的单元测试文件

自动扫描 `packages/` 目录下所有子包、所有 `*.ts` 工具模块（只考虑新增 `.ts` 文件），自动生成对应的单元测试文件。

- 自动生成文件：`packages/[子包名]/test/[模块名].spec.ts`

- 用 describe 定义测试用例，每个文件对应一个测试用例

- 用 test 定义测试用例，每个测试用例对应一个函数调用

- 用 expect 定义预期结果，每个测试用例对应一个 expect 断言

- 尽量覆盖函数中的所有分支、所有参数组合、所有异常情况

## 四、Agent 技能执行流程（标准流水线）

每次触发文档生成，Agent 必须按以下顺序完整执行：

1. 扫描 `packages/` 全量子包与 TS 源码文件，对比现有文档目录

2. 增量生成/更新/删除对应 `*.md` 模块文档

3. 自动生成并更新各子包 `index.md` 汇总首页

4. 自动同步更新 `config.mts` 导航、侧边栏配置

5. 保证 **源码、文档、站点配置** 三者完全同步

## 五、技能能力兜底说明

- 兼容未来新增任意子包：dom、shared、后续新增包全部自动适配

- 兼容单文件新增函数、函数修改、函数删除的增量更新

- 所有自动生成目录、文件、配置遵循项目原有规范，不新增自定义规则

- 严格遵循工程规范：仅识别 `packages` 目录发包源码，忽略 apps 应用层文件

- 忽略 `cache` / `node_modules` / `dist` 目录：仅对 `packages/` 目录下文件执行文档生成、配置同步能力

## 六、技能纠错机制

在生成上面的文档、单测文件时后，如果发现 `packages/[子包名]/src/[模块名].ts` 实现有明显严重错误，如语法错误、类型错误、逻辑错误等需告知用户。
